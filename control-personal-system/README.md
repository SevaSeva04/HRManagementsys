# HR System — Информационная система управления персоналом предприятия

Полнофункциональная веб-система для автоматизации кадровых процессов предприятия: учёт сотрудников, отделов, должностей, отпусков, отсутствий, кадровых перемещений и HR-аналитика.

## Функциональные возможности

### Аутентификация и авторизация
- JWT-аутентификация (access + refresh токены)
- 4 роли: Администратор, HR-менеджер, Руководитель, Сотрудник
- Разграничение прав доступа по ролям
- Хранение сессии в localStorage

### Управление сотрудниками
- Полная карточка сотрудника (ФИО, контакты, должность, отдел, оклад, руководитель)
- Поиск по ФИО, email, табельному номеру
- Фильтрация по отделу, должности, статусу
- Создание, редактирование, увольнение и восстановление сотрудников
- Экспорт списка сотрудников в CSV

### Организационная структура
- Справочник отделов с назначением руководителей
- Справочник должностей с вилками окладов
- Привязка должностей к отделам

### Кадровые процессы
- Приём на работу, перевод между отделами, изменение должности/оклада, увольнение
- Полная история всех кадровых изменений
- Временная шкала изменений в карточке сотрудника

### Отпуска и отсутствия
- Создание заявок на отпуск (черновик / на согласовании / одобрен / отклонён)
- Многоступенчатое согласование: руководитель → HR
- Проверка пересечения дат
- Подсчёт остатка отпускных дней
- Регистрация отсутствий: больничный, отгул, командировка

### Личный кабинет сотрудника
- Персональные данные и рабочая информация
- Баланс отпусков (визуальные прогресс-бары)
- История заявок и кадровая история
- Подача заявки на отпуск

### HR-аналитика и отчёты
- Дашборд: количество сотрудников, статусы, динамика найма
- Диаграммы: по отделам (Pie), по должностям (Bar), найм по месяцам
- Отчёты: по отделам, отпуска за период
- Экспорт в CSV

### Уведомления
- Внутренние уведомления при подаче/согласовании заявок
- Уведомления о кадровых изменениях
- Счётчик непрочитанных, отметка «прочитать все»

### Схема базы данных

**Основные таблицы:**
- `users` — учётные записи (логин, пароль, роль)
- `employees` — сотрудники (ФИО, контакты, должность, отдел, оклад)
- `departments` — отделы (название, руководитель)
- `positions` — должности (название, вилка окладов)
- `department_positions` — связь должностей и отделов
- `leave_requests` — заявки на отпуск (тип, даты, статус)
- `leave_balances` — баланс отпускных дней
- `absences` — отсутствия (тип, даты, причина)
- `employment_history` — история кадровых изменений
- `notifications` — внутренние уведомления

## Инструкция по запуску

### Вариант 1: Docker Compose (рекомендуется)

```bash
# Клонируйте репозиторий
git clone <url> && cd control-personal-system

# Запустите все сервисы
docker-compose up --build

# Откройте в браузере
# http://localhost:5173
```

### Вариант 2: Локальный запуск (для разработки)

**Предусловия:** Node.js 18+, PostgreSQL 15+

```bash
# 1. Запустите PostgreSQL (или через Docker)
docker run -d --name hr-postgres \
  -e POSTGRES_DB=hr_system \
  -e POSTGRES_USER=hr_admin \
  -e POSTGRES_PASSWORD=hr_secret_2024 \
  -p 5432:5432 postgres:15-alpine

# 2. Backend
cd backend
npm install
npx prisma migrate deploy
npx prisma db seed
npm run dev           # http://localhost:3001

# 3. Frontend (в другом терминале)
cd frontend
npm install
npm run dev           # http://localhost:5173
```

## Тестовые учётные записи

| Роль | Логин | Пароль | Доступ |
|------|-------|--------|--------|
| Администратор | `admin` | `admin123` | Полный доступ ко всем функциям |
| HR-менеджер | `hr` | `hr123` | Управление сотрудниками, отпусками, отчёты |
| Руководитель | `manager` | `manager123` | Сотрудники своего отдела, согласование отпусков |
| Сотрудник | `employee` | `employee123` | Личный кабинет, подача заявки на отпуск |
